# AI智能备考复习平台前端配置与启动说明

## 1. 需要分享的文件

团队成员需要拿到以下前端相关文件和目录：

```text
src/
index.html
package.json
package-lock.json
tsconfig.json
tsconfig.node.json
vite.config.ts
.gitignore
frontend-api.md
前端配置与启动说明.md
```

其中：

- `src/`：前端页面、样式、接口封装和类型定义。
- `package.json`：项目依赖和启动命令。
- `package-lock.json`：锁定依赖版本，保证大家安装出来的依赖一致。
- `vite.config.ts`：Vite 前端开发服务器配置。
- `frontend-api.md`：早期前后端接口联调说明，当前以仓库根目录 `docs/frontend.md`、`docs/backend.md` 和 `docs/build.md` 为准。

不需要分享以下内容：

```text
node_modules/
dist/
*.tsbuildinfo
```

这些内容可以通过命令重新生成，文件较大或属于构建缓存。

## 2. 环境要求

需要安装 Node.js 和 npm。建议使用较新的 Node.js 版本。

可以用下面命令检查：

```powershell
node --version
npm --version
```

## 3. 安装依赖

第一次拿到项目后，在项目根目录执行：

```powershell
cd C:\MyFile\mycode\SoftwareEngineering
npm install
```

如果团队成员的项目目录不同，把 `cd` 后面的路径换成自己的项目路径。

## 4. 启动前端

执行：

```powershell
npm run dev
```

启动成功后，浏览器访问：

```text
http://127.0.0.1:5173
```

## 5. 后端接口配置

前端默认请求后端地址：

```text
http://localhost:8000
```

如果后端也是本地启动，并且地址是 `http://localhost:8000`，不需要额外配置。

如果后端地址不同，可以在项目根目录新建 `.env` 文件：

```text
VITE_API_BASE_URL=http://你的后端地址
```

示例：

```text
VITE_API_BASE_URL=http://127.0.0.1:8000
```

修改 `.env` 后需要重新运行：

```powershell
npm run dev
```

## 6. 构建检查

提交或分享前建议执行一次：

```powershell
npm run build
```

如果命令成功，说明 TypeScript 和 Vite 构建检查通过。

## 7. 当前前端功能

当前前端已包含：

- 登录 / 注册页面
- 学习仪表盘
- 课程 / 考试目标管理
- 资料库上传与解析状态展示
- AI 知识提炼页面
- AI 问答页面
- AI 出题与自测页面
- 错题本页面
- 复习计划页面
- 管理员端预览页面

后端未启动时，页面会使用演示数据，方便展示界面效果；后端启动后，登录、目标、资料上传、AI 问答等功能应按 `docs/frontend.md` 和 `docs/backend.md` 中的接口进行联调。
